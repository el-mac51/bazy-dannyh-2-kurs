###### 1.

**Раздел "Instance" ("Экземпляр БД"):**

Раздел предназначен для управления конфигурацией и состоянием экземпляра MySQL Server. Информация и инструменты логически сгруппированы:

1. Dashboard (Панель управления):

Кнопки управления сервером: Start/Stop/Restart.

Индикатор текущего состояния сервера (Running/Stopped).

Быстрый доступ к логам и конфигурационным файлам.

2. Startup/Shut Down:

Настройки автозапуска сервера при загрузке ОС.

Параметры инициализации экземпляра.

3. Configuration:

Просмотр и редактирование файла my.cnf / my.ini.

Группы настроек: порты, пути к данным, буферы, логирование.

Применение изменений требует перезапуска сервера.

4. Users and Privileges:

Управление учетными записями пользователей.

Настройка глобальных и объектных привилегий.

Просмотр активных сессий и возможность их завершения.

5. Data Import/Restore и Data Export:

Импорт дампов (.sql, .zip) в существующие или новые схемы.

Экспорт схем или отдельных таблиц с гибкой настройкой объектов.

**Раздел "Performance" ("Производительность"):**

Раздел предоставляет инструменты мониторинга и анализа работы сервера в реальном времени.

1. Dashboard:

Графики нагрузки: соединения, запросы в секунду (QPS), использование CPU/RAM.

Индикаторы репликации (если настроена).

Возможность переключения между метриками.

2. Performance Reports:

Готовые отчеты: "Top 5 Slow Queries", "InnoDB Buffer Pool Usage", "Lock Contention".

Исторические данные для анализа трендов.

3. Status and System Variables:

Просмотр значений SHOW STATUS и SHOW VARIABLES.

Фильтрация и поиск по переменным.

Возможность изменения динамических переменных (с соответствующими правами).

4. InnoDB Engine Monitoring:

Детальная статистика по буферному пулу, транзакциям, блокировкам.

Информация о файлах tablespace и фрагментации.

5. Query Statistics:

Анализ выполненных запросов: время, количество вызовов, полные сканирования.

CREATE TABLE `simpledb`.`resume` (

  `idresume` INT NOT NULL AUTO_INCREMENT,

  `user_id` INT NOT NULL,

  `title` VARCHAR(100) NOT NULL,

  `skills` TEXT,

  `created` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP,

  PRIMARY KEY (`idresume`),

  CONSTRAINT `fk_resume_users`

    FOREIGN KEY (`user_id`)

    REFERENCES `simpledb`.`users` (`id`)

    ON DELETE CASCADE

    ON UPDATE CASCADE

) ENGINE=InnoDB DEFAULT CHARSET=utf8mb3;

  

ON DELETE CASCADE — при удалении пользователя автоматически удаляются все его резюме из таблицы resume

ON UPDATE CASCADE — при изменении id пользователя автоматически обновляются все значения user_id в таблице resume

Это обеспечивает ссылочную целостность и предотвращает появление "осиротевших" записей

Ограничение целостности (foreign key) предотвращает появление "осиротевших" записей. Вставка возможна только при наличии соответствующего id в родительской таблице users. Это гарантирует, что каждое резюме принадлежит реально существующему пользователю.

Error Code: 1452. Cannot add or update a child row: a foreign key constraint fails (`simpledb`.`resume`, CONSTRAINT `fk_resume_users` FOREIGN KEY (`user_id`) REFERENCES `users` (`id`) ON DELETE CASCADE ON UPDATE CASCADE)
